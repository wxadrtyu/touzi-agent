V3.2 核心设计原则

1

投资建议优先

五段式产业链研究是生产投资建议的前置过程，最终产出仍是投资建议

2

少即是多

4 个 Agent 而非 7 个，五段式单次调用，CoT 自我辩论

3

上下文质量 > 流程严谨

80% 精力投入 Context 层，15% Prompt 层，5% Guardrails

4

市场差异在 Context 层

Agent 核心逻辑市场无关，差异由 Market Adapter + 配置注入

5

快速反馈

拒绝原因即时回写 Context，比任何 Guardrails 都有效

Agent 1: 五段式 Prompt 模板（单次调用）

关键改进：五段式作为单个 Prompt 模板，1 次 LLM 调用完成全部五段。不是 5 次独立调用。LLM 在做红队证伪（第三段）时能直接回看产业链拆解（第一段）的全部推理过程，零信息丢失。

五段流程

  1. 产业链拆解（C0-C4）：终端需求 → 系统集成 → 核心部件 → 材料工艺 → 上游资源
  2. 证据复筛：候选公司逐个检查，标注 A/B/C/D 证据等级
  3. 红队证伪：反向审查 5 个标准问题
  4. 防幻觉检查：拎出可能夸大的结论
  5. 综合判断：底层逻辑三问 + DCF 估值 + 投资建议 + 跟踪表

Agent 2: CoT 自我辩论

关键改进：单 Agent CoT 自我辩论替代双 IA 分 Agent。同一个 Agent 在完整上下文中"先正方再反方再综合"，零信息损失。LLM 的 CoT 能力就是为此设计的。

CoT 三步

第一步：正方论证

IA#1 视角：论证为什么 Sicong 应该执行这个建议

第二步：反方反驳

IA#2 视角：逐条审查漏洞，必须提出至少 3 个实质性反驳

第三步：综合判断

基于正反双方观点，给出共识等级 + 最终建议

共识等级

🟢 高共识：立即推送，强推 🟡 有条件共识：推送 + 附加条件 🟠 存在争议：Sicong 裁决 🔴 强烈反对：拦截待观望

4. Context 层 — RAG 检索与上下文装配

RAG 检索流程

用户问题 / 标的分析请求 ↓ 向量检索 (ChromaDB) + 结构化查询 (SQLite) ├─ 财报切片（A级） ├─ 电话会纪要（A/B级） ├─ 研报切片（C级） ├─ 舆情切片（D级） ├─ 财务因子查询 └─ L7 历史操作查询 ↓ 混合 Re-rank (Gemini Flash) ├─ 语义相关性评分 0-10 ├─ 证据等级标注（A/B/C/D） └─ 时间衰减 ↓ 上下文装配（按场景选择装配矩阵） ↓ 长度控制（Tier 1/2/3 压缩）→ 注入 Prompt

上下文装配矩阵

场景注入内容长度预算
深度研究L2 全部事实 + L4 产业链图谱 + L3 画像 + L7 历史 + L5 拒绝原因100K tokens
实时咨询L2 核心事实 + L5 当前估值 + L6 当前假设 + L3 画像30K tokens
持仓监控L6 阈值 + L2 最新事实 + 实时行情10K tokens
趋势预测L2 历史因子 + L4 宏观乘数 + L3 大佬画像20K tokens

快速反馈循环（V3.2 新增）

最重要的改进：每次 Sicong 拒绝提案后，拒绝原因即时回写 Context。下次同类标的分析时注入："上次你建议买入 XXX，Sicong 拒绝了，原因是'{reason}'。本次分析时请注意避免类似问题。" 这个反馈循环比任何 Guardrails 都有效。

证据等级标注（V3.2: 无硬编码权重）

初版不设任何软引导，完全交给 LLM 判断。保留配置入口 evidence_weights.yaml 供后期添加（初版 enabled: false）。

等级含义处理方式
A公告 / 年报核心证据，可直接作结论
B调研 / 管理层交流辅助判断，需交叉验证
C研报 / 媒体仅参考，需 A/B 级支撑
D社交媒体仅线索，不能作结论（叙事领袖除外）

5. Harness 层 — 工作流·Guardrails

Guardrails 分级（V3.2: 硬阻塞 + 软警告）

关键改进：只有数学硬规则阻塞，不会误判。软警告让 Sicong 有更多信息做判断。真正的安全网是 Sicong 确认。

Guardrail类型行为理由
第一道：溯源校验🟡 软警告不拦截，标注 ⚠️ 引用待核验引用格式多变，正则误判率高
第二道：交易一致性🔴 硬阻塞拦截，强制改 HOLD数学硬规则，安全边际不能妥协
第三道：WACC/g 边界🔴 硬阻塞拦截，要求重算数学硬规则，估值参数不能超范围
第四道：防幻觉🟡 软警告不拦截，附防幻觉检查报告判断题不是数学题，LLM 标注比正则拦截更有用

慢循环工作流

触发：快循环异动 / Sicong 主动请求 / 财报发布 ↓ 候选筛选（Python 量化，0 Token） ↓ Agent 1: 研究分析（五段式单次调用） ↓ Agent 2: 决策（CoT 自我辩论单次调用） ↓ Guardrails 检查（2 硬阻塞 + 2 软警告） ↓ Sicong 确认门禁 ├─ 确认 → Agent 4: 交易执行 └─ 拒绝 → Agent 3: 拒绝原因分析 → 回写 Context ↓ Agent 3: 操作记录 + 纪律评分

6. 业务生命周期设计

未买前：预测与准入

  • 买什么：量价/流动性初筛 + 安全边际
  • 为什么：五段式研究提炼核心假设
  • 怎么买：买入击球区 + 建仓步长

已买后：追踪与预警

  • 知行合一追踪：持仓状态自动转换
  • 锁定假设警报线 (L6)
  • 假设证伪 → 操作预警

卖出操作

  • 止损触发：跌破止损线
  • 假设断裂：财报指标跌破红线
  • 止盈触发：超过估值中枢 30%+
  • 仓位失衡：单股仓位超上限

紧急熔断例外：价格跌破止损线超过 5%，无需共识投票，直接推送红色硬止损警报。

7. Sicong 强纠错保护机制

高风险操作二次确认

默认选项为"取消执行"：

  • 追高：高于估值中枢 15%+
  • 超仓：单股仓位超上限
  • 亏损加仓：亏损 > 8%
  • 极度恐慌：VIX > 30

个人画像盲区检测

  • 资金规模、风险边界、持仓上限
  • 投资风格、历史盲区清单
  • 首次启动对话式问卷采集
  • AI 发现新盲区 → 画像更新提案

操作纪律评分

  • 遵循系统建议 → +1 分
  • 被警告仍坚持 → -1 分
  • 定期报告评分与改进建议
