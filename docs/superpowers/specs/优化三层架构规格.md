将 Agent 按其主要活跃层映射到同心圆模型，每个 Agent 都有 6P Prompt 定义、依赖的 Context 管道、受 Harness 约束。

```

     ↕ 调度与约束
【Context 中层】RAG 混合检索 + 情绪数据 + L1-L7 装配矩阵
     ↕ 信息供给
【Prompt 内层】7 个 Agent 的 6P 大脑
     ├─ 研究咨询 Agent (原 Deep RAG 增强)
     ├─ 情报分析 Agent (Sentiment)

```

**优点**：最贴合同心圆模型，三层职责清晰，每层可独立优化
**缺点**：Agent 跨层协作需要明确定义接口

## 方案 B：功能域 + 三层横切

按功能域组织（研究域/决策域/执行域/反思域），三层架构作为横切关注点贯穿每个域。

```
研究域: Deep RAG + Sentiment     ← Prompt/Context/Harness 各有定义
决策域: Audit Council + IA#1 + IA#2
执行域: 富途 MCP + Guardrails
反思域: 操作分析 Agent
```

**优点**：业务流程清晰，域内高内聚
**缺点**：三层架构变成附属说明，不够突出，与同心圆模型契合度低

---

## 方案 C：双循环架构

区分"快循环"（实时咨询/查询）和"慢循环"（定时深度分析/提案），每个循环内嵌三层架构。

```
慢循环 (System 2): 定时触发 → 全 Agent 链 → 共识门禁 → 提案推送
```

**优点**：实时性与深度分析兼顾
**缺点**：两个循环的 Context/Harness 需要分别维护，复杂度高

### 4.2 混合检索管道（Hybrid Search Pipeline）

```
用户 Query / Agent 请求
        │
    ┌───┴───┐
    ▼       ▼
[Dense 语义]  [Sparse 关键词+数字]
 BAAI/bge-    SQLite FTS5
 small-zh-    全文索引
 v1.5 (384维)  关键词+数字
    │              │
 Top K1=15     Top K2=15
    └───┬───┘
        ▼
  [RRF 互惠等级融合]
   RRF_Score = 1/(60+r_dense) + 1/(60+r_sparse)
        │
   融合 Top 15
        ▼
  [Re-rank 重排]
   Gemini Flash 评分 0-10
   阈值拦截 < 5分
        │
   精炼 Top N (N≤8, ≤6k tokens)
        ▼
  注入 Agent Context
```

**关键设计**：
- **双路召回**：语义检索（防稀释）+ 关键词精确数字检索（防数字丢失）
- **RRF 融合**：避免两路打分范围不同无法加权
- **轻量 Re-rank**：Gemini Flash 快速评分，拦截噪声

## 1. 这个设计效果会好吗？——坦率说，过度工程化了

**核心问题：当前 V3.1 把 80% 的设计精力投入到了只影响 5% 输出质量的地方。**

LLM 输出质量的权重分布，根据实际经验：

```
上下文质量 ████████████████████████████████████████ 80%
Prompt 设计 ████████████ 15%
流程/Guardrails ███ 5%
```

当前 V3.1 的问题：

| 过度工程化 | 实际影响 |
|---|---|
| 7 个 Agent 串联，每次提案要跑 5+ 次 LLM 调用 | 慢、贵、且 Agent 间信息传递有损 |
| 五段式流程作为独立工作流阶段 | LLM 单次调用内就能完成五段式，拆开反而丢失上下文连贯性 |
| 证据等级加权 1.0/0.7/0.4/0.1 | 拍脑袋数字，LLM 自己能判断证据质量，不需要硬编码权重 |
| 4 道 Guardrails 全部阻塞式 | 拦截太多会拖慢迭代，真正的安全网是 Sicong 确认 |
| 双 IA 分两个 Agent 辩论 | 单个 Agent 内用 CoT 自我辩论效果更好，上下文不丢失 |

### 更好的策略：少即是多

**核心思路：减少 Agent 数量，增加上下文质量，让 LLM 发挥推理能力。**

#### 策略 A：精简 Agent 链（推荐）

```
当前 7 Agent 串联 →  精简为 4 Agent

Agent 1: 研究分析 Agent（合并研究咨询+情报+审计）
  ├─ 五段式研究 → 作为 Prompt 模板，不是独立工作流阶段
  ├─ L0-L4 拆解 → 单次调用内完成
  ├─ 三脑审计 → CoT 内多视角切换（"先以白毛股神视角看..."）
  └─ 输出：L5 提案 + 证据表 + 红队表 + 防幻觉表

Agent 2: 决策 Agent（合并 IA#1 + IA#2）
  ├─ CoT 自我辩论："先论证为什么该买，再论证为什么不该买，最后综合判断"
  ├─ 比分两个 Agent 效果更好——上下文完整，不丢失信息
  └─ 输出：操作建议 + 共识等级（自评置信度）

Agent 3: 操作分析 Agent（不变）
Agent 4: 交易执行 Agent（富途 MCP，不变）
```

**为什么 CoT 自我辩论比双 Agent 辩论好？**
- 双 Agent：IA#1 生成提案 → 序列化为文本 → IA#2 读取文本 → 反驳。**信息有损传递**。
- CoT 自我辩论：同一个 Agent 在完整上下文中"先正方再反方再综合"。**零信息损失**，且 LLM 的 CoT 能力就是为此设计的。

