# 股票与宏观多级漏斗投研 Agent 系统设计规范

- **日期**: 2026-06-14
- **状态**: Proposed (v1.5 — 移除达里奥，L3 大脑库确认为白毛股神+孙宇晨，架构升级为双 IA 共识门禁)
- **核心定位**: 针对 **Sicong** 的私人投资教练 Agent。主动给出"买什么、为什么、怎么买、买了后做什么、变化后做什么、何时卖"的全生命周期决策，并通过**双 Investment Advisor 内部辩论**过滤低质建议，再推送给 Sicong 确认。

---

## 1. 核心定位与业务生命周期设计

### 1.1 未买前：预测与准入 (Pre-Purchase)
- **买什么**：候选股必须通过量价/流动性初筛，且估值具备高安全边际。
- **为什么**：大模型提炼核心投资假设（Thesis）记录于 L5。
- **怎么买**：计算安全边际，设定"买入击球区"与资金建仓步长。
- **数据落盘**：提炼出核心假设与合理买入边界，记录在 `L5_valuation.md` 的 `## 投资前假设与建仓方案` 中。

### 1.2 已买后：追踪与预警 (Post-Purchase)
- **知行合一追踪**：Sicong 录入交易后，个股自动转为"持仓中 (Holding)"状态。
- **锁定假设警报线 (L6)**：锁定买入时承诺的财务假设，如营收增速必须 > 15%、毛利率降幅不超过 2%。
- **买了后做什么**：设定止盈止损策略，持续监控财报与电话会。
- **出现变化做什么**：假设被证伪或估值飘移超限时，输出明确的操作预警与建议（加仓/减仓/清仓）。

### 1.3 卖出操作完整设计 (Sell Operations)
卖出分**系统主动触发**与 **Sicong 主动发起**两类，均经过共识门禁（见第 3 节）。

**系统主动触发卖出的 4 种场景：**

| 触发类型 | 触发条件 | 建议操作 |
|---|---|---|
| **止损触发** | 价格跌破设定止损线（如 -12%） | 建议立即清仓 |
| **假设断裂** | 财报指标跌破买入红线（如营收增速 < 10%） | 建议减仓/清仓 |
| **止盈触发** | 价格超过估值中枢 30%+，进入高估区间 | 建议分批止盈 |
| **仓位失衡** | 大涨导致单股仓位超出上限 | 建议减仓至上限内 |

**紧急熔断例外**：价格跌破止损线超过 5%，无需共识投票，直接推送红色硬止损警报，要求 Sicong 即时确认。

| # | Agent | 职责 | 模型 |

## 2.1 RAG 知识库设计（Deep RAG Analyst 的数据来源）

RAG 知识库分为两层存储：
- **向量库（ChromaDB）**：存所有长文本（财报/纪要/研报），用于语义检索
- **结构化数据库（SQLite / JSON）**：存财务因子、持仓快照、宏观指标，用于精确查询

### 七大类数据来源：

| 类别 | 具体内容 | 获取方式 | 更新频率 |
|---|---|---|---|
| **1. 公司基本面文件** | 10-K 年报、10-Q 季报、8-K 重大事件、**财报电话会议纪要（优先级最高）**、投资者日演示文稿、Proxy Statement | SEC EDGAR API / `_inbox/` 手动投喂 | 每季度 |
| **2. 管理层言论** | CEO/CFO 公开访谈全文、公司新闻稿、重大人事变动公告、股东信全文 | 新闻爬取 / 手动投喂 | 不定期 |
| **3. 产业链与竞争情报** | 行业研究报告、竞争对手对比分析、**供应链上下游依赖图谱（白毛股神风格）**、专利与技术路线图 | 手动投喂 + AI 提炼 → 写入 L4 | 不定期 |
| **4. 宏观与政策文件** | 美联储 FOMC 会议纪要、CPI/GDP/PMI 报告、产业政策与监管变化（AI 监管、芯片出口限制等） | 官方 API / RSS | 每月 |
| **5. 机构与专业分析** | 卖方研究报告（GS/MS 等）、13F 持仓披露（巴菲特/Duquesne 最新持仓）、做空报告（反向参考） | SEC EDGAR API / 手动投喂 | 每季度 |
| **6. 实时舆情与社交信号** | 白毛股神推文存档、关键财经 KOL 观点、Reddit 热点摘要、财经新闻全文（非标题） | Twitter API / RSS / Sentiment Agent | 每日 |
| **7. 系统历史知识（元 RAG）** | L7 历史操作日志（让 IA 检索"上次类似情况结果如何"）、历史复盘报告、过往提案决策记录 | L7 自动写入 | 实时 |

### 关键设计原则：
- **财报电话会纪要优先级高于财报本身**：管理层在 Q&A 环节的回答比数字更能反映真实状况与风险。
- **`_inbox/` 作为万能投喂口**：Sicong 把任何 PDF/网页/截图拖入此文件夹，系统自动解析分类入库。
- **元 RAG 实现经验传承**：IA 给建议前自动检索历史相似案例，避免重蹈覆辙。

## 3. 共识门禁机制（Consensus Gate）

**任何提案在推送给 Sicong 之前，必须先经过双 Investment Advisor 内部辩论，达成共识方可推出。**

### 共识流程：
```
Agents 1-4 产出分析结果
          ↓
Investment Advisor #1（主提案方）
  整合所有情报生成完整建议草案
          ↓
Investment Advisor #2（挑战方 · 魔鬼代言人）
  逐条审查 IA#1 的逻辑漏洞：
    · 假设是否成立？有没有忽略的风险？
    · 白毛股神视角：这家公司真的是卡脖子件吗？
    · 孙宇晨视角：这是叙事泡沫还是真实基本面驱动？
    · 是否与 Sicong 历史盲区冲突？
          ↓
      达成共识？→ 推送 Sicong 确认
```

### 四级共识等级与推送策略：

| 共识等级 | 条件 | 推送行为 | 标签 |
|---|---|---|---|
| 🟢 **高共识** | IA#1 与 IA#2 完全同意 | 立即推送，强烈建议执行 | `强推` |
| 🟡 **有条件共识** | 同意但 IA#2 附加补充条件 | 推送，同时展示 IA#2 的附加条件与风险提示 | `有条件同意` |
| 🟠 **存在争议** | IA#2 有重大异议 | 两方观点同时呈现给 Sicong，标注分歧焦点，Sicong 做最终裁决 | `存在争议` |
| 🔴 **IA#2 强烈反对** | IA#2 发现致命逻辑缺陷 | 暂不推送，等待新数据窗口（如下季财报）重新评估 | `拦截待观望` |

**卖出操作同样经过双 IA 辩论**——不是系统单方面说卖就推，IA#2 审查通过才推送 Sicong 确认。

## 4. L7 操作记录与复盘总结机制

每一笔操作（买/卖/加/减仓）均写入 L7，格式如下：

```markdown
# 操作记录 AAPL-BUY-2026-06-14

## 基本信息
- 操作类型：买入 / 加仓 / 减仓 / 清仓
- 标的：AAPL  价格：$148  数量：100股  仓位占比：8%
- 共识等级：🟢 高共识（3/3 支持）

## 决策背景
- 系统建议：分两批买入（首批 5%，第二批 3%）
- Sicong 实际决定：一次性买入 8%（未遵循分批建议）
- 违规标记：❗ 未按建议执行（记入操作纪律评分）

## 买入时锁定的假设红线 (L6)
- Q3 营收增速必须 > 10%
- 毛利率不低于 43%
- 止损线：$138

## 复盘评价（定期自动触发）
- 3个月后价格：$175（+18.2%）
- 假设是否被证伪：否
- 操作评价：盈利，但未遵循分批建仓纪律
- 经验/教训：本次操作验证了"低估值+财报超预期"策略有效，但一次性全仓存在择时风险，下次建议坚持系统的分批建议
- 新增画像条目：Sicong 在有信心时倾向于一次性重仓，不遵循分批纪律
```

### 定期复盘总结（每季度/每次平仓触发）：
- 本季度所有操作盈亏汇总
- 遵循系统建议 vs 自行决策的胜率对比
- 新归纳的经验与教训
- 自动追加进 `sicong.md` 的"已验证认知"与"已验证盲区"

## 5. Sicong 强纠错保护机制（核心）

### 高风险操作二次确认门禁（默认选项为"取消执行"）：
- 追高：价格高于估值中枢 15%+ 仍要买入
- 超仓：单股仓位超出设定上限
- 亏损加仓：持仓亏损 > 8% 还选择加仓
- 极度恐慌期操作（VIX > 30）

### Sicong 个人画像盲区检测（`L3-profile/sicong.md`）：
- 包含：资金规模、风险边界、持仓上限、投资风格、历史盲区清单
- 首次启动时由 Investment Advisor 主动发起**对话式问卷**采集
- AI 发现新盲区时自动生成"画像更新提案"，Sicong 确认后追加

### 操作纪律评分系统：
- 每次遵循系统建议 → +1分
- 每次被警告仍坚持 → -1分，记录案例及后续结果
- 定期向 Sicong 报告"本月操作纪律评分"与改进建议

## 7. L1-L7 分层记忆与决策架构映射

| 层级 | 定位名称 | 本项目数据载体与映射 | 写入权限 |
|---|---|---|---|
| **L1** | 原始痕迹层 (Raw) | 财报 PDF 切片、K线数据、大佬访谈原文 | 全自动 |
| **L2** | 原子事实层 (Facts) | 标准财务因子、宏观指标明细 | 全自动 |
| **L3** | 认知画像层 (Profiles) | `sicong.md` + `buffett.md` + `druckenmiller.md` + `serenity.md` + `justin_sun.md`（共 5 个大脑，达里奥已移除） | 人工授权（提案） |

## 9. Web 控制台核心功能

1. **每日早报推送**：Investment Advisor 主动汇报今日 3 件最重要的事（需处理的提案/市场异动/持仓预警）。
2. **共识标签可视化**：每张提案卡片显示 🟢🟡🟠🔴 共识等级及各大脑的投票原因。
3. **卖出建议专栏**：独立展示止损/止盈/假设断裂触发的卖出建议，明确呈现"系统建议 vs 你的成本价 vs 当前市价"三者关系。
4. **操作纪律看板**：展示 Sicong 本月操作纪律评分及历史遵从率趋势图。
5. **画像盲区警报区**：Sicong 个人盲区防线检测结论固定显示在审批页顶部，优先于大佬审计意见。

## 10. 系统验证与测试方案
- **单元测试**：量价过滤脚本、共识投票聚合逻辑、L6 假设断裂检测
- **联调测试**：冷启动画像建立对话→模拟买入→模拟假设断裂→触发卖出建议→共识流程→Sicong 拒绝→系统记录拒绝到 L7 Mistake Log
- **安全测试**：模拟 Sicong 发起超仓操作，验证二次确认门禁拦截及默认选项为"取消"
