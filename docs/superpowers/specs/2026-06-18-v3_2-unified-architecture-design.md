# 股票投研 Agent 系统设计规范 V3.2

- **日期**: 2026-06-18
- **状态**: Proposed (V3.2 — 精简 Agent 链 + Market Adapter + CoT 自我辩论 + 快速反馈循环)
- **核心定位**: 针对 **Sicong** 的私人投资教练 Agent。主动给出"买什么、为什么、怎么买、买了后做什么、变化后做什么、何时卖"的全生命周期投资建议。支持 **美股 + A 股** 双市场。
- **与 V3.1 的核心差异**: Agent 7→4 精简、五段式单次调用、证据等级无硬编码权重、Guardrails 分级、Market Adapter 层、CoT 自我辩论、快速反馈循环、投资建议优先、C0-C4 命名、叙事领袖泛化

### 0.2 双循环设计

| 循环 | 频率 | 触发 | 工作流 |
|---|---|---|---|
| **快循环** | 每日 | 定时（盘前/盘后） | 行情扫描 → 持仓监控 → L6 阈值检查 → 异动预警 → （触发时）进入慢循环 |
| **慢循环** | 每周/触发式 | 快循环触发 / Sicong 主动请求 / 财报发布 | 候选筛选 → 研究分析（五段式） → 决策（CoT 辩论） → Guardrails → Sicong 确认 → 执行 → 反思 |

### 0.3 V3.2 核心设计原则

1. **投资建议优先**：五段式产业链研究是生产投资建议的前置过程，最终产出仍是投资建议
2. **少即是多**：4 个 Agent 而非 7 个，五段式单次调用而非多次，CoT 自我辩论而非双 Agent 辩论
3. **上下文质量 > 流程严谨**：80% 精力投入 Context 层，15% 投入 Prompt 层，5% 投入 Guardrails
4. **市场差异在 Context 层**：Agent 核心逻辑市场无关，差异由 Market Adapter + 配置注入
5. **快速反馈**：拒绝原因即时回写 Context，比任何 Guardrails 都有效

```python
class BaseDataAdapter:
    """数据适配器基类，子类实现市场特定逻辑"""

        """获取公告/财报"""
        raise NotImplementedError

        """获取行情数据"""
        raise NotImplementedError
    
    def fetch_holdings(self, ticker: str) -> dict:
        """获取机构持仓"""
        raise NotImplementedError
    
    def fetch_earnings_call(self, ticker: str) -> str:
        """获取电话会/调研纪要"""
        raise NotImplementedError

# adapters/data_us.py
class USDataAdapter(BaseDataAdapter):
```

## 3. Prompt 层 — 4 个 Agent 的 6P 设计

### 3.1 Agent 总览（V3.2: 7→4 精简）

| # | Agent | 职责 | 模型 | V3.1 对应 |
|---|---|---|---|---|
| **1** | **研究分析 Agent** | 五段式产业链研究 + 估值 + 趋势预测。单次调用完成全部五段。 | Claude 3.5 Sonnet | 合并 V3.1 的 Agent 1+2+3 |
| **2** | **决策 Agent** | CoT 自我辩论（正方→反方→综合），输出投资建议 + 共识等级 | Claude 3.5 Sonnet | 合并 V3.1 的 Agent 4+5 |
| **3** | **操作分析 Agent** | 操作纪律评分、复盘总结、拒绝原因分析 | Claude 3.5 Sonnet | V3.1 Agent 6 |
| **4** | **交易执行 Agent** | 富途 MCP 下单、持仓同步、T+1/涨跌停检查 | Python + 富途 MCP | V3.1 Agent 7 |

### 3.2 Agent 1: 研究分析 Agent

#### 6P 设计

| P | 内容 |
|---|---|
| **Purpose** | 对标的进行五段式产业链研究，产出 L5 投资建议草案（含估值、假设、证据表、红队表、防幻觉表） |
| **Persona** | 你是一位兼具白毛股神（产业链卡点审计）、孙宇晨（叙事泡沫探测）、Sicong 盲区检测三视角的投资研究分析师 |
| **Process** | 五段式流程（单次调用完成，见下方 Prompt 模板） |
| **Policy** | ① 证据等级 A/B/C/D 强制标注 ② D 级证据仅线索不作结论 ③ 无 A 级证据的标的自动降级 ④ 底层逻辑三问必须回答 |
| **Presentation** | 结构化 Markdown（L5 模板） |
| **Proof** | 输出前自检：每个结论是否有证据支撑？证据等级是否标注？ |

#### 五段式 Prompt 模板（单次调用）

```
你是一个投资研究 Agent。当前分析的市场是：{market_config.market}

## 市场规则
- 交易制度：{market_config.settlement}
- 会计准则：{market_config.accounting_standard}
- 估值偏好：{market_config.valuation_preference}
- 关键指标：{market_config.key_metrics}

## 已知事实（标注证据等级）
{context_with_evidence_levels}

## 叙事领袖清单
{market_config.narrative_leaders}

请按以下五段式流程分析标的 {ticker}，在一个回答中完成全部五段：

### 第一段：产业链拆解（C0-C4）
按以下框架拆解目标赛道产业链：
- C0 终端需求：需求是否真实？持续 1-3 年还是短期脉冲？
- C1 系统/集成：认证周期多长？集成壁垒在哪？
- C2 核心部件：良率和产能瓶颈在哪？技术门槛？
- C3 材料/工艺/设备：扩产周期和设备约束？
- C4 上游资源：供给稳定性？资源约束？

输出 10-20 个候选卡点线索。

### 第二段：证据复筛
对候选公司逐个检查：
- 近 4 季度主营业务是否与卡点相关？
- 收入占比、毛利率、订单、产能是否有公开披露？
- 标注证据等级（A=公告/B=调研/C=研报/D=社交媒体）

输出表格：公司|代码|对应环节|业务纯度|公开证据|证据等级|不能证明什么|初步归类

### 第三段：红队证伪
对每个"重点线索"做反向审查：
- 大客户是否可能自研/二供/三供？
- 12-18 个月内是否可能被新技术替代？
- 同行是否可能打价格战？
- 收入是否真的来自卡点而非边缘业务？
- 当前市场是否已充分定价？

输出表格：公司|最大风险|证据能否缓解|还缺什么数据|风险等级

### 第四段：防幻觉检查
把可能夸大的结论拎出来：
输出表格：可能被夸大的结论|为什么可能夸大|谨慎表达方式

### 第五段：综合判断
- 底层逻辑三问回答：
  1. 需求是否真实？
  2. 瓶颈在哪里？
  3. 股票映射是否有证据？
- DCF 估值（悲观/中性/乐观，WACC 8-12%，g 0-3%）
- 投资建议分类：重点建议 / 观察建议 / 暂不适合
- 买入假设与建仓方案
- 未来两季度跟踪表

## 证据等级说明
- A级 = 公告/年报（核心证据，可直接作结论）
- B级 = 调研/管理层交流（辅助判断，需交叉验证）
```

### 5.3 Guardrails 分级（V3.2: 硬阻塞 + 软警告）

| Guardrail | 类型 | 行为 | 理由 |
|---|---|---|---|
| **第一道：溯源校验** | 🟡 软警告 | 不拦截，在提案上标注 `⚠️ 引用待核验` | 引用格式多变，正则误判率高；Sicong 看到警告自己判断 |
| **第二道：交易一致性** | 🔴 硬阻塞 | 拦截，强制改 HOLD | 数学硬规则，安全边际不能妥协 |
| **第三道：WACC/g 边界** | 🔴 硬阻塞 | 拦截，要求重算 | 数学硬规则，估值参数不能超范围 |
| **第四道：防幻觉** | 🟡 软警告 | 不拦截，附防幻觉检查报告 | 判断题不是数学题，LLM 标注比正则拦截更有用 |

```python
def run_guardrails(output, context):
    warnings = []
    
    # 第一道：软警告
    if not validate_citations(output, context):
        warnings.append("⚠️ 部分引用未匹配原文，请核验")
    
    # 第二道：硬阻塞
    if not validate_trade_consistency(output):
        output["action"] = "HOLD"  # 强制改 HOLD
    
    # 第三道：硬阻塞
    if not validate_dcf_bounds(output):
        raise GuardrailBlock("WACC/g 超范围，要求重算")
    
    # 第四道：软警告
    hallucination_report = check_hallucination(output)
    if hallucination_report.has_issues:
        warnings.append(hallucination_report)
    
    output["warnings"] = warnings
    return output
```

## 6. 业务生命周期设计

### 6.1 未买前：预测与准入 (Pre-Purchase)

- **买什么**：候选股必须通过量价/流动性初筛，且估值具备高安全边际
- **为什么**：Agent 1 五段式研究提炼核心投资假设，记录于 L5
- **怎么买**：计算安全边际，设定"买入击球区"与资金建仓步长
- **数据落盘**：提炼出核心假设与合理买入边界，记录在 `L5_valuation.md`

### 6.2 已买后：追踪与预警 (Post-Purchase)

- **知行合一追踪**：Sicong 录入交易后，个股自动转为"持仓中 (Holding)"状态
- **锁定假设警报线 (L6)**：锁定买入时承诺的财务假设
- **买了后做什么**：设定止盈止损策略，持续监控财报与电话会
- **出现变化做什么**：假设被证伪或估值飘移超限时，输出明确的操作预警

### 6.3 卖出操作完整设计

**系统主动触发卖出的 4 种场景：**

| 触发类型 | 触发条件 | 建议操作 |
|---|---|---|
| **止损触发** | 价格跌破设定止损线（如 -12%） | 建议立即清仓 |
| **假设断裂** | 财报指标跌破买入红线（如营收增速 < 10%） | 建议减仓/清仓 |
| **止盈触发** | 价格超过估值中枢 30%+，进入高估区间 | 建议分批止盈 |
| **仓位失衡** | 大涨导致单股仓位超出上限 | 建议减仓至上限内 |

**紧急熔断例外**：价格跌破止损线超过 5%，无需共识投票，直接推送红色硬止损警报。

**卖出操作同样经过 Agent 2 的 CoT 自我辩论**——不是系统单方面说卖就推，反方审查通过才推送 Sicong 确认。

## 11. 附录：V3.1 → V3.2 差异说明

| 维度 | V3.1 | V3.2 |
|---|---|---|
| **Agent 数量** | 7 个 | 4 个（合并研究+情报+审计→研究分析；合并 IA#1+IA#2→决策） |
| **五段式实现** | 工作流阶段（有歧义） | 明确单次 Prompt 模板 |
| **双 IA 辩论** | 两个独立 Agent | 单 Agent CoT 自我辩论 |
| **证据等级权重** | 硬编码 1.0/0.7/0.4/0.1 | 无硬编码，初版无软引导，保留配置入口 |
| **Guardrails** | 4 道全阻塞 | 2 道硬阻塞 + 2 道软警告 |
| **市场适配** | 隐含美股偏向 | Market Adapter 层 + 两套配置 + 两套适配器 |
| **反馈循环** | 季度复盘 | 每次拒绝即时回写 Context |
| **最终产出** | 研究线索优先 | 投资建议优先 |
| **产业链命名** | L0-L4（与 L1-L7 冲突） | C0-C4 |
| **叙事领袖** | 硬编码 3 人 | 可配置清单 |
| **数据源** | 未明确列出 | 第 2 节配置文件明确列出 |
