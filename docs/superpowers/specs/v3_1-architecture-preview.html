🔬 产业链卡点研究方法论 V3.1 核心

研究哲学

核心原则：先拆产业链，再找卡点，最后映射公司。

  1. 研究线索优先：最终目标是输出研究线索，不是投资建议
  2. 证据强制溯源：所有关键数字必须写来源、日期和证据等级。无来源标"待核验"
  3. 公司分类标准：重点线索 / 观察线索 / 暂不适合作为案例

底层逻辑三问

问题核心检验点
1. 需求是否真实？需求来自谁？是否有公告/财报/招标/资本开支/政策支撑？短期脉冲还是持续 1-3 年？
2. 瓶颈在哪里？哪一层扩产最慢？替代最难？认证周期最长？良率/设备/材料/工艺构成硬约束？
3. 股票映射是否有证据？哪些公司真实覆盖卡点？有收入/订单/客户/产能/认证/募投证据？主营业务还是蹭热点？

📊 证据等级体系 V3.1 新增

等级来源类型定位使用规则
A级公告、年报、半年报、招股书、交易所互动、招标/中标公告核心证据可作为结论支撑
B级机构调研、管理层交流、权威行业数据、客户供应链公开信息辅助判断可辅助判断，不可单独作结论
C级券商研报、媒体报道、行业文章参考线索只能参考，需 A/B 级交叉验证
D级社交媒体、KOL、论坛、传闻线索只能作线索。例外：老黄/孙哥/特朗普发言可作有效线索

📋 五段式研究流程 V3.1 核心

第一段：产业链拆解 — 先画地图，不先列股票
按 L0-L4 拆解目标赛道，输出 10-20 个候选卡点线索
↓
第二段：证据复筛 — 用公告和财报穿透概念
逐个检查候选公司，输出证据表（含证据等级）
↓
第三段：红队证伪 — 主动反驳
对每个"重点线索"做反向审查，输出红队反证表
↓
第四段：防幻觉检查 — 把可能夸大的结论拎出来
输出防幻觉检查表
↓
第五段：最终输出 — 适合短视频展示
产业链卡点地图 + 证据表 + 红队表 + 跟踪表 + 防幻觉表

L0-L4 产业链拆解框架

层级定义关注点
L0 终端需求最终应用场景需求真实性
L1 系统/集成系统方案、整机、平台、交付、认证、客户导入认证周期、客户壁垒
L2 核心部件决定性能或交付的关键部件、技术、良率、产能良率、产能瓶颈
L3 材料/工艺/设备材料、工艺、设备、检测、国产化、扩产周期扩产周期、设备约束
L4 上游资源原材料、能源、资源、基础设备供给稳定性

四级共识等级（V3.1 受证据等级影响）

等级条件推送行为
🟢 高共识IA#1 与 IA#2 完全同意，且核心证据 ≥ A 级立即推送，强烈建议执行
🟡 有条件共识同意但附加条件，或核心证据为 B 级推送 + 展示附加条件
🟠 存在争议IA#2 有重大异议，或核心证据仅 C 级两方观点同时呈现，Sicong 裁决
🔴 强烈反对致命缺陷，或核心证据不足/仅 D 级暂不推送，等待新数据窗口
