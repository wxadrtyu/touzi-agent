## 1. 系统概述

### 1.1 设计目标
1. **以期望值为核心**：全系统围绕 `Expectancy`（以 R 为单位）优化，而非胜率。
2. **量化引擎为决策内核**：所有数字、仓位、风险由 Python 计算；LLM 永不输出未经计算的数字。
3. **自主策略发现 + 诚实验证**：LLM 跨完整 edge 菜单持续提出策略假设，经回测筛选 + 前向纸面交易认证后晋级，按已证明的期望值分配资金。
4. **行为矫正与能力成长**：以"行为期望值缺口"量化 Sicong 偏离计划的代价，持续教练，使其成长为优秀交易者。
5. **三市场适配**：US 为主，HK 次之，A 股作为特殊处理的小仓位 sleeve。

### 1.2 核心设计原则（Approach A：系统化内核 + LLM 研究/教练）
| 原则 | 说明 |
|---|---|
| **期望值优先** | 优化 `Expectancy = (Win%×Avg_Win_R) − (Loss%×Avg_Loss_R)`，胜率是虚荣指标 |
| **数字归代码** | Python 拥有一切数字与风险决策；LLM 拥有研究、判断、教练 |
| **小亏大赢** | 风格 = 快速止损 + 让利润奔跑（低胜率、高期望值，本质如此） |
| **前向验证为认证闸门** | 真实资金前必须通过前向纸面交易；回测仅用于规则类策略的筛选与学习 |
| **护栏硬阻塞** | 风险上限、引用校验为硬规则；默认选择即纪律选择 |
| **演进式晋级** | 任何策略 candidate→probation→promoted→retired，资金随已证明的 edge 缩放 |

### 1.3 个人约束（Sicong）
- **资金**: 约 HK$1,300,000（基准货币 HKD）
- **市场**: US（主）+ HK + A 股；可选 crypto/其他
- **风格**: 主动交易，每月数笔；快速止损，优质/有潜力股持有数月以上（系统已定义为 "Trend Core + Tactical Satellite"）
- **关注度**: 每日数次，可对盘中告警做出反应
- **风险偏好**: 单笔风险 0.5–1% 权益、组合总热度 ≤ ~6%（可调）

## 2. 架构总览

四层 + 编排循环。原则：**Python 拥有每一个数字与风险决策；LLM 拥有研究、判断与教练；人工门禁守在任何资金变动之前。**

```
┌─────────────────────────────────────────────────────────────┐
│  人工门禁  — 每份提案由 Sicong 确认 / 调整 / 拒绝             │
└─────────────────────────────────────────────────────────────┘
        ▲ 提案(期望值/仓位/止损/风险)        │ 决策
        │                                     ▼
┌─────────────────────────────────────────────────────────────┐
│  编排层 — 4 个循环                                            │
│  • 快循环(每日/盘中): 持仓监控、止损、告警                    │
│  • 慢循环: 研究 → 决策提案                                    │
│  • 发现循环: 策略提出/测试/晋级                               │
│  • 复盘循环: 操作后日志 + 行为教练                            │
└─────────────────────────────────────────────────────────────┘
     ▲                   ▲                      ▲
┌──────────────┐  ┌────────────────────┐  ┌──────────────────┐
│ 量化引擎      │  │ LLM 层(我)          │  │ 记忆 + RAG        │
│ (Python)     │  │ • 策略假设生成      │  │ • L1–L7 记忆      │
│ • 信号        │◄►│ • 定性论证 + 红队   │◄►│ • 结构化库(数字)  │
│ • 期望值      │  │ • 交易复盘          │  │ • 向量库(文档+记忆)│
│ • 仓位/风险   │  │ • 行为教练          │  │ • Vault(可读真源) │
│ • 回测/前向   │  │ (永不输出数字)      │  └──────────────────┘
│ • 执行准备    │  └────────────────────┘          ▲
└──────────────┘                          ┌──────────────────┐
     ▲                                     │ 市场适配器        │
     └──────────────────────────────────  │ US / HK / CN      │
                                           └──────────────────┘
```

**单次决策数据流**：
1. **摄取**：数字进结构化库，文档进向量库，统一由 L1–L7 模型索引。
2. **量化引擎**对当前数据运行**已验证策略** → 产出候选交易，每笔以**期望值**评分、由**风险模型**预先定仓（含止损与组合风险检查）。
3. **LLM 层**为每个候选补充定性论点、红队证伪，并**检索 Sicong 自身相似历史交易及其结果**。
4. **装配提案**：交易 + 期望值 + 仓位 + 止损 + 组合风险影响 + 定性论证 + 行为标记 → 人工门禁。
5. **执行**（纸面或经富途 MCP 实盘）→ 写入 **L7 日志** → **复盘循环**评分纪律、回灌 RAG 记忆与发现循环。

**底层双轨验证**：规则类策略 = 回测 + 前向；自由裁量/LLM 选股 = **仅前向**。任何东西在通过前向测试与人工批准前都不接触真实资金。

## 3. 核心指标：期望值与 R 单位

每笔交易 `R = 进场价 − 止损价`（承担的风险金额）。盈亏以 R 的倍数表达，使所有策略可比：

```
策略期望值(R) = (Win% × Avg_Win_R) − (Loss% × Avg_Loss_R)
```

示例：40% 胜率、均赢 +3R，60% 败率、均亏 −1R → 期望值 = 0.4×3 − 0.6×1 = **+0.6R/笔**（低胜率却强正期望）。**这是全系统优化的数字。**

- 所有回测/前向结果均**净成本**报告（佣金/滑点/点差/印花税等），禁止毛期望值。
- 期望值带**置信区间**评估，不看单点数字。

## 4. 量化策略引擎（Python）

### 4.1 策略库 — 统一接口
每个策略是同构模块，便于引擎统一运行/测试/比较：
- **Hypothesis**（为何有 edge）
- **Universe**（适用市场/过滤）
- **Entry rules** / **Exit rules**（止损 + 移动止盈/目标）
- **Holding horizon**
- **Status**: `candidate → probation → promoted → retired`

### 4.2 仓位 — 风险优先、期望值加权
```
基础仓位 = (权益 × 单笔风险%) / (进场价 − 止损价)
```
- `单笔风险%` 起始 0.5–1%（封顶单笔亏损）。
- 以**分数 Kelly（封顶，如 ¼-Kelly）**按策略**已证明**的期望值缩放：高 edge 策略更大仓，未证明策略给最小仓。
- 波动率归一（ATR），使不同波动标的承担可比风险。

### 4.3 组合风险模型 — 硬性护栏（阻塞，非警告）
| 控制项 | 规则（可调） | 理由 |
|---|---|---|
| **总热度** | 未平仓风险之和 ≤ ~6% 权益 | 限制全员触损时的总损失 |
| **单仓上限** | ≤ ~15–20% 权益 | 单标的不可拖垮账户 |
| **相关/集群上限** | 限制相关标的/板块的合计敞口 | 5 只 AI 股 = 1 个赌注 |
| **最大回撤降险** | 组合回撤达 −X% 时减小仓位、暂停新仓 | 活过坏 regime |
| **单市场上限** | US/HK/CN 各设上限 | 隔离单市场冲击 |
| **FX 敞口** | 跟踪并可选封顶 USD/CNY/HKD 集中度 | 避免货币意外集中 |

### 4.4 成本始终内置
每次回测与前向均含佣金、滑点、点差与市场特定成本（A 股印花税/过户费等）。期望值只以**净成本**报告。

## 5. 策略发现与前向验证循环

替代 V3.2 被污染的"回测即认证"路径。

### 5.1 生命周期
```
  IDEA           SCREEN            PROBATION          PROMOTED          RETIRED
(LLM 提出)  →  (回测,仅规则类) →  (前向纸面,小仓) →  (实盘小仓→按edge → (edge 衰减
                                                      缩放, 人工批准)    或违规)
```

### 5.2 假设生成（自主发现）
LLM 跨完整 edge 菜单持续生成策略假设——趋势/动量、质量、价值、事件驱动、季节性、均值回归、跨资产、情绪等任何有**经济学依据**者——并参考 Sicong 自身的有效结果。每个假设写成 §4.1 标准模块。

### 5.3 验证闸门（统计诚实）
| 闸门 | 要求 |
|---|---|
| **经济学依据** | 必须有"为何有效"，禁止盲目数据挖掘 |
| **回测筛选**（仅规则类） | 在**样本外/walk-forward**数据上净成本正期望 |
| **最小样本** | 样本足够，不在 n=5 上晋级 |
| **置信而非点估计** | 期望值连同置信区间为正 |
| **回撤可接受** | 最差回撤在容忍范围内 |
| **低相关** | 为现有组合带来新东西 |
| **前向证明** | 通过前向纸面交易方可接触真实资金 |

### 5.4 防过拟合
- 仅以 **walk-forward / 样本外** 评判，绝不在拟合期评判。
- **参数节俭**：惩罚复杂重调参，偏好简单稳健。
- **多重检验意识**：因会提出大量策略，部分会因偶然好看，故相应提高门槛。
- **双轨**：规则类 = 回测 + 前向；自由裁量 LLM 选股 = 仅前向。

### 5.5 晋级、缩放与淘汰
- **晋级实盘**始终需 Sicong 批准。
- **资金随 edge 缩放**（在 §4.3 风险上限内）。
- **衰减检测**：监控滚动期望值；edge 侵蚀或违规则自动标记降级/退役。Edge 会死，系统预期并主动淘汰。

## 6. RAG 与记忆

### 6.1 三存储，职责清晰
| 存储 | 拥有 | 用于 |
|---|---|---|
| **结构化库**(SQLite/Parquet) | 一切**数字**：价格/因子/财务/收益/期望统计/交易记录 | 精确查询、信号、回测、风险模型 |
| **向量库**(ChromaDB) | 一切**文档 + 记忆**：财报/纪要/笔记/日志 | 研究与教练的语义检索 |
| **Vault**(Obsidian/Markdown) | 人类可读规范文件 | Sicong 读写；L3/L5/L7 真源 |

**铁律**：数字永不来自向量库。若信号/风险数字被语义检索到，即为 bug。

### 6.2 映射 L1–L7
- **L1 原始 / L2 事实** → 数字进结构化库，文档切片进向量库（标注 `source/date/ticker/market/evidence_level`）。
- **L3 画像 / L4 关联 / L5 判断** → Vault（规范）+ 嵌入检索。
- **L6 跟踪规则** → 结构化库（引擎自动检查阈值）。
- **L7 交易日志** → 结构化库（交易记录，供期望计算）**+** 向量库（理由/教训，供教练）。双写是刻意的：L7 既是数据也是记忆。

### 6.3 四个集合
1. **研究语料** — 财报、纪要、调研、新闻。
2. **个人交易记忆（最高价值）** — 决策、被拒想法及原因、复盘/教训。
3. **策略库** — 各策略假设 + 其测试结果。
4. **市场/板块上下文** — 宏观、regime 观察。

### 6.4 检索管线
**混合**（语义 + 关键词/BM25）→ 元数据过滤（`date` point-in-time、`ticker`、`market`、`doc_type`、`evidence_level`）→ **re-rank** → 预算内装配。
**硬规则**：每条事实带 `source`+`date`；**无法解析的引用硬失败**；历史分析**必须**应用 `date` 过滤（point-in-time 干净）。

### 6.5 最高价值查询
任何新提案前运行：*"检索我自身相似 setup / 同标的 / 同策略的历史交易及其结果与教训。"* 此检索驱动 §7 行为教练——让 RAG 复利的是 Sicong 的**能力**，而非仅研究。

## 7. 行为教练与硬性护栏

### 7.1 两类
- **硬性护栏** = 硬阻塞，默认拒绝，保护资金免于灾难性错误。
- **教练** = 软性教育反馈，识别模式、量化代价、提升能力。

### 7.2 硬性护栏（默认阻塞）
| 规则 | 行为 |
|---|---|
| **必须止损** | 无止损不接受进场 |
| **禁止亏损加仓** | 对亏损仓位加仓被阻塞 |
| **禁止超仓** | 任何突破 §4.3 上限的仓位被拒 |
| **禁止破组合上限** | 破热度/集中/相关上限的交易被拒 |
| **连亏冷却** | 连续 N 次止损或日内亏损阈值后暂停新仓（杀复仇交易） |
| **仅已验证策略** | 验证策略之外的交易需显式、留痕的 override |

Override 存在但**有摩擦且留痕**——可覆盖，但是刻意记录的行为，系统追踪其表现（通常更差，眼见即教训）。默认即纪律选择。

### 7.3 核心指标：行为期望值缺口
```
行为缺口 = 计划期望值(R) − 实际实现期望值(R)
```
即**因偏离计划而泄漏的 edge**（过早止盈、超止损硬扛、盈后超仓、FOMO 进场）。以 R 与港币双报。示例：*"本季策略产生 +0.7R/笔，你实现 +0.3R；−0.4R 缺口 = HK$X，几乎全部来自止损后多扛 2–3 日。"*

### 7.4 模式检测 + 教练循环
从 L7 + RAG 检测复发行为模式（过早砍赢家、损失厌恶硬扛、盈后过度自信加仓、过度交易）：盘前标记、盘后评分、周期性趋势报告。

### 7.5 能力成长追踪
显式度量是否**变好**：行为缺口是否缩小？复发错误是否减少？将"成长为优秀交易者"目标可量化。

## 8. 市场适配器（US / HK / CN）

### 8.1 核心市场无关
策略、期望值、仓位、风险在核心；各市场以**配置 + 适配器**注入差异。在 V3.2 的 US/CN 上新增 **HK**。

### 8.2 各市场配置
| | **US** | **HK** | **A 股(CN)** |
|---|---|---|---|
| 结算 | T+0 | T+2 | **T+1** |
| 涨跌停 | 无 | 无 | **±10% / ±20%** |
| 做空 | 允许 | 受限 | 受限 |
| 成本 | 佣金 | 佣金 + 印花税 | 佣金 + **印花税** + 过户费 |
| 数据 | SEC EDGAR / Yahoo | HKEX / 富途 | 巨潮 / 东方财富 |
| 时段/日历 | US | HK | CN |

### 8.3 关键洞察：A 股规则会**打破**核心风险模型
风格 = 快速止损，但 A 股：
- **T+1**：当日买入不可当日卖出；立刻走坏的进场要到次日才能退出。
- **跌停**：−10% 常封死无买盘，**止损物理上无法成交**，可能被困并次日继续跳空。

因此风险模型将 A 股仓位视为带 **gap/流动性风险**：真实风险不是 `entry−stop`，而可能远大于此。引擎处理：
- **A 股仓位更小**（计入可能无法成交的止损）；
- **建模最坏 gap 风险**（假设 1–2 个跌停无法退出）纳入期望/风险计算；
- 临近涨跌停或带已知催化（财报/解禁）的 A 股候选**标记为高风险**。

US/HK 无涨跌停，"快速止损"可干净执行——这是核心仓位向 US/HK 倾斜、A 股作为更小特殊处理 sleeve 的结构性理由。

### 8.4 适配器接口（扩展 V3.2）
每市场实现：**数据适配器**（fetch）、**交易适配器**（带规则检查执行：T+1/涨跌停/做空）、**成本模型**（喂入每次期望计算）。可用处经富途 MCP 路由。

### 8.5 货币
账户基准 **HKD**；持仓跨 USD/CNY/HKD，系统跟踪并报告 **FX 敞口**，可选上限防止单一货币意外集中。

1. **规则类策略**：样本外/walk-forward 回测筛选 → 前向纸面交易 → 实盘小仓 → 按已证明期望值缩放。

4. **持续监控**：滚动期望值、回撤、相关、行为缺口；衰减即降级/退役。
