## 1. 系统概述

### 1.1 设计目标
1. **时间归一化期望值为核心**：优化年化 R（期望值 × 机会频率），而非每笔期望值或胜率。
2. **量化引擎为数字内核**：所有数字、仓位、风险由 Python 计算；LLM 永不输出未经计算的数字。
3. **四管道并行供给**：量化（带/不带 LLM 校验）、纯 LLM、决策支持各自独立产出可执行策略/建议。
4. **诚实验证**：回测仅用于规则类策略筛选；一切真实资金前须经前向纸面交易；按已证明战绩缩放。
5. **跑赢基准**：成功 = 净成本、风险调整后**跑赢统一基准**；否则不如持有指数。
6. **行为矫正与成长**：以"行为期望值缺口"与"override 质量"量化并改进 Sicong 的交易行为。

### 1.2 核心设计原则
| 原则 | 说明 |
|---|---|
| **时间归一化期望值优先** | 优化年化 R，在风险/路径约束下；胜率是虚荣指标 |
| **数字归代码** | Python 拥有一切数字与风险决策；LLM 拥有研究、判断、教练 |
| **小亏大赢** | 快速止损 + 让利润奔跑（低胜率、高期望值，本质如此） |
| **跑赢或持有指数** | 跑不赢基准就该持有被动指数；基准是硬标尺 |
| **前向验证为认证闸门** | 真实资金前必须前向纸面交易；回测仅筛选规则类 |
| **护栏硬阻塞** | 风险上限、引用校验为硬规则；默认即纪律选择 |
| **演进式晋级** | 任何策略/管道 candidate→probation→promoted→retired，资金随已证明 edge 缩放 |

## 2. 核心目标与指标

### 2.1 R 单位与期望值
每笔 `R = |进场价 − 止损价|`。盈亏以 R 倍数表达。
`每笔期望值(R) = (Win% × Avg_Win_R) − (Loss% × |Avg_Loss_R|)`。

### 2.2 时间归一化（修复 V4.0 缺陷 D1）
**主优化指标 = 年化期望值** ≈ `每笔期望值(R) × 年化交易频率`。低频但漂亮的策略不得仅凭每笔期望值晋级。

### 2.3 风险/路径约束与基准（修复 D2/D3）
- 期望值**在风险约束下**最大化，而非"唯一指标"：晋级须同时满足回撤、连亏、方差门槛。
- 报告**增长/风险调整指标**（MAR = 年化收益 / 最大回撤；或 SQN）。
- **成功标准 = 净成本、风险调整后跑赢统一全局基准**（默认：以美股为主的全球指数，如 QQQ/ACWI，实现期定）。
- **被动核心仓建议**：建议保留一部分资金于被动指数（"跑不赢就持有"），既是基准也是默认资金归宿。

### 2.4 一切净成本
期望值/收益仅以**净成本**报告：佣金、**滑点**、点差、市场特定费（HK 印花税）、**FX 兑换成本与税**（如美股股息预扣）。禁止毛期望值（修复 D13）。

## 3. 架构总览：四管道 + 横切纪律层

四条**独立**管道，各自可单独向 Sicong 交付可执行策略/建议；全部汇入同一**横切的"行为纪律 + 人工门禁 + L7 日志"层**，再经统一的前向验证与按战绩缩放。底层由"量化引擎 / LLM 层 / 记忆+RAG / 市场适配器"四个共享层支撑。

```
         ┌──────────── 四条独立管道（各自产出，互不依赖）────────────┐
         │                                                          │
 Pipeline 1（量化工作流，白盒因子引擎）                              │
   1-1:  量化信号 ─▶ LLM 校验/叠加 ─▶ 可执行策略 ┐                   │
   1-2:  量化信号 ───────────────────▶ 可执行策略 ┤                   │
 Pipeline 2（纯 LLM 工作流，黑盒）                ├─▶               │
   RAG + 大模型 ──────────────────────▶ 可执行策略 ┤                   │
 Pipeline 3（决策支持，按需）                     │                   │
   用户自定义查询 ─▶ RAG+大模型分析/校验 ─▶ 针对性建议 ┘                │
         │                                                          │
         └──────────────────────────┬───────────────────────────────┘
                                     ▼
        ┌──────────── 横切层：行为纪律 + 人工门禁 + L7 日志 ─────────┐
        │ 硬性护栏(阻塞) · 行为期望值缺口 · override质量 · 模式检测   │
        │ → 人工确认/筛选/调整/拒绝 → 写 L7 → 前向验证 + 按战绩缩放    │
        └────────────────────────────────────────────────────────────┘
                                     ▲ 共享支撑
   ┌──────────┐  ┌──────────────┐  ┌──────────────────┐  ┌──────────┐
   │ 量化引擎  │  │ LLM 层(我)    │  │ 记忆 + RAG        │  │ 市场适配 │
   │ (Python) │  │ 研究/推理/校验 │  │ 结构化库/向量库   │  │ US / HK  │
   └──────────┘  └──────────────┘  └──────────────────┘  └──────────┘
```

### 3.1 四管道职责
| 管道 | 节奏 | 性质 | 输入 | 产出 | 验证 |
|---|---|---|---|---|---|
| **1-1 量化+LLM校验** | 慢 | 白盒量化 + LLM 叠加 | 因子信号 + RAG/上下文 | 可执行策略 | walk-forward 回测 + 前向 |
| **1-2 纯量化** | 慢 | 白盒量化 | 因子信号 | 可执行策略 | walk-forward 回测 + 前向 |
| **2 纯 LLM** | 慢（每日定时） | 黑盒 RAG+LLM | RAG + 行情 + 新闻/研报 | 可执行策略（带推理） | **仅前向** + 复盘 |
| **3 决策支持** | 快（实时按需） | RAG+LLM 答询（**独立**） | Sicong 的查询（针对其**自主手动交易**） | 针对性可执行建议 | 采纳后表现 + 行为缺口度量 |

> **Pipeline 3 独立性**：用于 Sicong **不依赖本系统信号、自行下单**时的按需咨询；与 Pipeline 1/2 **无过程依赖**，仅与 Pipeline 2 **共享 RAG+LLM 引擎**。

### 3.2 LLM 权限边界（按管道）
- **1-1**：LLM 作为量化信号的**校验/叠加层**——只能**否决或缩小（veto/shrink），绝不放大**量化引擎给出的仓位/信号。
- **1-2**：**无 LLM**，量化信号直达交付（作为 1-1 的对照基线）。
- **2**：LLM **自主黑盒生成**策略候选；这是独立产出，不触碰量化信号，故"绝不放大"不适用——它走自己的前向认证 + 按战绩缩放。
- **3**：LLM 针对 Sicong 的查询做分析/校验并给建议；不自动下单，只供决策。
- 一致铁律：**不以 LLM 自身回测认证任何策略**（我知道历史，回测被污染）。2 与 1-1 的 LLM 部分仅以**前向战绩**认证、按战绩缩放。

### 3.3 汇流与统一验证
四管道的任何产出都不直达资金：统一经**横切纪律层**（§8）→ 人工门禁（§10）→ 写 L7 → **前向验证 + 按已证明战绩缩放**（§5.5/§5.6）。这保证无论哪条管道，真实资金都只跟随已被前向证明的 edge。

### 3.4 1-1 vs 1-2：内建 A/B 实证
量化管道**并存**带 LLM 校验（1-1）与不带（1-2）两路，对**同一批量化信号**分别前向跟踪，直接度量"LLM 叠加是否真的提升期望值/降低回撤"。若 1-1 长期不优于 1-2，则 LLM 校验层降权或移除——用证据而非信念决定 LLM 的价值。

## 4. 量化引擎（Python）

### 4.1 策略统一接口
同构模块：`Hypothesis` / `Universe` / `Entry rules` / `Exit rules`（**止损 + 移动止损/目标**）/ `Holding horizon` / `Status`（candidate→probation→promoted→retired）。**必须支持横截面策略**（同时看多标的做排序，如相对动量/热门板块续强）——见 §12。

### 4.2 仓位 — 风险优先、期望值加权
`基础仓位 = 权益 × 单笔风险% / |进场价 − 止损价|`。
- 单笔风险% 起始 0.5–1%。
- **分数 Kelly（封顶 ¼）** 按策略**已证明**的期望值缩放；未证明给最小仓；**带最小样本门槛**。
- 波动率归一（ATR）。
- **最小经济仓位**：低于"成本吃掉 edge"的下限不开仓（修复 D11）。

### 4.3 组合风险模型 — 硬性护栏（阻塞）
| 控制项 | 规则（可调） |
|---|---|
| 总热度 | 未平仓风险之和 ≤ ~6% 权益 |
| 单仓上限 | ≤ ~15–20% 权益 |
| 相关/集群上限 | 限制相关标的/板块合计敞口；**集群定义具体化**（行业 GICS + 滚动收益相关阈值二者取严，修复 D12） |
| 最大回撤降险 | 组合回撤达 −X% 时减小仓位、暂停新仓（**按市值实时回撤**，§12） |
| 单市场上限 | US/HK 各设上限 |
| FX 敞口 | 跟踪并可选封顶 USD/HKD 集中度 |
| 并发策略上限 | 同时在跑策略数上限（避免资金/成本碎片化，修复 D11） |
| Regime 分散 | 主动跨 regime 分散（趋势/均值回归/波动），不押单一 regime（修复 D7） |

> 风险模型对**所有管道**的最终下单提案统一生效（横切层调用）。

### 4.4 成本始终内置
见 §2.4。回测/前向每笔含佣金、滑点、点差、市场费、FX、税。

## 5. 策略发现与前向验证循环

### 5.1 生命周期
`IDEA(LLM 提/规则) → SCREEN(回测,仅规则类) → PROBATION(前向纸面,小仓) → PROMOTED(分层实盘,按 edge 缩放,人工批准) → RETIRED(衰减/违规)`。
四管道产出统一走此生命周期；其中 Pipeline 2 的黑盒策略**跳过回测 SCREEN，仅前向**。

### 5.2 假设生成
LLM 跨**全 edge 菜单**持续生成假设——趋势/动量、质量、价值、事件驱动、季节性、均值回归、跨资产、情绪等任何有**经济学依据**者，并参考 Sicong 自身有效结果。每个写成 §4.1 模块。

### 5.3 验证闸门（统计诚实）
经济学依据 ✓；**样本外/walk-forward 净成本正期望** ✓；**最小样本** ✓；**置信区间为正**（bootstrap，§12）✓；**回撤/路径可接受** ✓；**低相关** ✓；**前向证明** ✓；**年化 R 达标** ✓。

### 5.4 防过拟合
仅以 walk-forward/样本外评判；**参数节俭**；**多重检验意识**；**预注册 + 前向加权**（承认 LLM 提案带后视偏差，重前向轻回测，修复 D10）；双轨（规则类=回测+前向；纯 LLM=仅前向）。

### 5.5 部署吞吐（修复 D4）
**并行前向纸面**多策略同跑；**分层实盘**（过前向即极小仓上实盘，CI 收窄则放大）；**偏好快认证策略**；低频策略接受更宽 CI + 更小仓。

### 5.6 晋级、缩放、淘汰
晋级实盘须 Sicong 批准；资金随 edge 缩放（§4.3 上限内）；**衰减检测**监控滚动期望值，edge 侵蚀/违规即降级或退役。

## 6. RAG 与记忆

### 6.1 三存储
| 存储 | 拥有 | 用于 |
|---|---|---|
| 结构化库(SQLite/Parquet) | 一切数字：**复权后**价格/因子/财务/收益/期望统计/交易记录 | 精确查询、信号、回测、风险 |
| 向量库(ChromaDB) | 一切文档+记忆：财报/纪要/笔记/日志 | 研究与教练语义检索 |
| Vault(Obsidian/MD) | 人类可读规范文件 | Sicong 读写；L3/L5/L7 真源 |

**铁律**：数字永不来自向量库。**复权铁律**：价格类一律复权后入库（修复 D9）。

### 6.2 L1–L7 映射
L1/L2 → 数字进结构化库、文档进向量库（标注 source/date/ticker/market/evidence_level）；L3/L4/L5 → Vault + 嵌入；L6 → 结构化库阈值；L7 → 结构化库（交易记录）+ 向量库（理由/教训）。

### 6.3 集合
1. 研究语料（财报/纪要/调研/新闻）2. **个人交易记忆（最高价值）** 3. 策略库（假设+测试结果）4. 市场/板块/regime 上下文。

### 6.4 检索管线
混合（语义+BM25）→ 元数据过滤（**date point-in-time**/ticker/market/doc_type/evidence_level）→ re-rank → 预算内装配。**无法解析引用硬失败**；历史分析必须 date 过滤。

### 6.5 最高价值查询
任何新提案前：检索"我自身相似 setup/同标的/同策略历史交易及结果与教训"，驱动 §8 行为教练。

## 7. Pipeline 2 与 3 — RAG+LLM 工作流（共享引擎）

两条管道共享同一 RAG+LLM 引擎，但职责不同、互不依赖。

### 7.1 Pipeline 2 — 纯 LLM 选股（黑盒，主动产出策略）
- **每日定时聚合**：RAG 档案 + 实时行情 + 财经新闻 + 研报。
- **产出**：可投 ticker 排序清单 + **完整推理逻辑**（论点、证据等级、风险、止损建议），作为可执行策略候选。
- **人工筛选**：Sicong 从清单中挑选；选中者进入横切纪律层 + 人工门禁 + 前向跟踪。
- **认证与缩放**：仅以**前向战绩**认证，按已证明期望值缩放真实资金。
- **诚实定位**：黑盒选股历史上是最不可靠的 edge 源，故按战绩而非信念定仓；人工筛选 + §8 纪律层为其 over-trust 的缓冲。

### 7.2 Pipeline 3 — 决策支持（独立按需 Q&A）
- **场景**：Sicong **不依赖系统信号、打算自行手动下单**时，向系统提问。
- **流程**：用户自定义查询 → RAG + 大模型分析/校验 → 输出**针对性、可执行**的建议（基于 RAG + 当前持仓 + 风险模型 + 历史教训），标注证据等级与风险。
- **边界**：**不自动下单**，只供 Sicong 自主决策；与 Pipeline 1/2 无过程依赖。
- **纳入纪律**：Sicong 据此实际下的手动单，仍应记入 L7 并由 §8 横切纪律层覆盖（前提：登记计划 R，见 §8）。

## 8. 横切层：行为纪律 + 人工门禁（覆盖所有交易，含手动）

> 行为教练**不是一条管道**，而是**所有交易**（来自 Pipeline 1/2 信号，或 Sicong 凭 Pipeline 3 建议/完全自主下的手动单）在"采取行动"边界上必经的横切层。Pipeline 3 是独立的 Q&A 管道（§7.2），**不属于**本层。
>
> **手动交易纳入前提**：Sicong 的手动单须登记**计划 R**（进场/止损/目标）才能纳入行为缺口计算与教练，否则只记录不评分。

### 8.1 两类
硬性护栏（硬阻塞，§8.2）+ 教练（软性教育，量化代价、提升能力）。

### 8.2 硬性护栏（默认阻塞）
必须止损；禁止亏损加仓；禁止超仓（破 §4.3 上限）；禁止破组合上限；**连亏冷却**（连续 N 次止损/日内亏损阈值后暂停新仓）；**仅已验证策略**（例外需显式留痕 override）。Override 有摩擦且留痕，系统追踪其表现。

### 8.3 行为期望值缺口
`行为缺口 = 计划期望值(R) − 实际实现(R)`，以 R 与 HKD 双报，定位泄漏来源（过早止盈、超止损硬扛、盈后超仓、FOMO）。

### 8.4 override 质量评分（修复 D8）
不一概视偏离为泄漏。每次 override 标注理由并事后评估：**合理 override**（有据/留痕/事后正确）→ 正面，沉淀新认知；**冲动 override**（无据/违纪/事后错误）→ 负面，沉淀盲区。目标是培养**判断力**而非盲从。

### 8.5 覆盖手动/自主交易
本层覆盖一切**已登记**的交易，包括 Sicong 凭 Pipeline 3 建议或完全自主下的手动单：对其同样执行硬性护栏（如有经系统下单）、记入 L7、计算行为缺口（需登记计划 R）、纳入模式检测与教练。目的：系统的纪律不被"手动后门"绕过。

### 8.6 成长追踪
显式度量：行为缺口是否缩小？复发错误是否减少？override 质量是否提升？将"成长为优秀交易者"可量化。

## 12. 工程保真要求（回测/执行不得失真）
**强制实现要求**（源自代码评审，避免回测乐观偏差）：
1. **风险模型并入回测**：经 §4.3 组合风险模型，不得无约束并发持仓。
2. **跳空/滑点成交**：止损/目标在跳空穿越时按**更差价**成交；每笔计滑点；进场按**真实可成交价**（次开/收）。
3. **移动止损/动态退出**：支持随价上移的移动止损与动态退出（让利润奔跑）。
4. **横截面支持**：策略 `generate` 须能同时接收多标的、做排序。
5. **按市值回撤**：逐日浮动权益计回撤。
6. **bootstrap 置信区间**：应对小样本与肥右尾。
7. **复权价**：见 §6.1。
8. **健壮性**：除零保护；Kelly/晋级最小样本门槛；做空可行性按市场规则校验。
