## 1. 系统概述

### 1.1 设计目标
1. **时间归一化期望值为核心**：优化年化 R（期望值 × 机会频率），而非每笔期望值或胜率。
2. **量化引擎为数字内核**：所有数字、仓位、风险由 Python 计算；LLM 永不输出未经计算的数字。
3. **三管道协作**：白盒量化策略、黑盒discretionary选股、实时决策辅助并行供给。
4. **诚实验证**：回测仅用于规则类策略筛选；一切真实资金前须经前向纸面交易；按已证明战绩缩放。
5. **跑赢基准**：成功 = 净成本、风险调整后**跑赢统一基准**；否则不如持有指数。
6. **行为矫正与成长**：以"行为期望值缺口"与"override 质量"量化并改进 Sicong 的交易行为。

### 1.2 核心设计原则
| 原则 | 说明 |
|---|---|
| **时间归一化期望值优先** | 优化年化 R，在风险/路径约束下；胜率是虚荣指标 |
| **数字归代码** | Python 拥有一切数字与风险决策；LLM 拥有研究、判断、教练 |
| **小亏大赢** | 快速止损 + 让利润奔跑（低胜率、高期望值，本质如此） |
| **跑赢或持有指数** | 跑不赢基准就该持有被动指数；基准是硬标尺 |
| **前向验证为认证闸门** | 真实资金前必须前向纸面交易；回测仅筛选规则类 |
| **护栏硬阻塞** | 风险上限、引用校验为硬规则；默认即纪律选择 |
| **演进式晋级** | 任何策略/管道 candidate→probation→promoted→retired，资金随已证明 edge 缩放 |

## 2. 核心目标与指标

### 2.1 R 单位与期望值
每笔 `R = |进场价 − 止损价|`。盈亏以 R 倍数表达。
`每笔期望值(R) = (Win% × Avg_Win_R) − (Loss% × |Avg_Loss_R|)`。

### 2.2 时间归一化（修复 V4.0 缺陷 D1）
**主优化指标 = 年化期望值** ≈ `每笔期望值(R) × 年化交易频率`。两个 +0.6R/笔的策略，若一个年触发 2 次、一个年触发 40 次，价值天差地别。低频但漂亮的策略不得仅凭每笔期望值晋级。

### 2.3 风险/路径约束与基准（修复 D2/D3）
- 期望值**在风险约束下**最大化，而非"唯一指标"：晋级须同时满足回撤、连亏、方差门槛。
- 报告**增长/风险调整指标**（MAR = 年化收益 / 最大回撤；或 SQN）。
- **成功标准 = 净成本、风险调整后跑赢统一全局基准**（基准默认：以美股为主的全球指数，如 QQQ/ACWI，实现期定）。
- **被动核心仓建议**：建议保留一部分资金于被动指数（"跑不赢就持有"），既是基准也是默认资金归宿。

### 2.4 一切净成本
期望值/收益仅以**净成本**报告：佣金、**滑点**、点差、市场特定费（HK 印花税；A 股印花税/过户费——暂缓）、**FX 兑换成本与税**（如美股股息预扣）。禁止毛期望值（修复 D13）。

│  →walk-forward 回测筛选   →产出可投 ticker + 完整推理逻辑     │

### 3.1 三管道职责
| 管道 | 节奏 | 性质 | 产出 | 验证 |
|---|---|---|---|---|
| **P1 量化** | 慢循环 | 白盒规则 | 候选量化策略 | walk-forward 回测 + 前向纸面 |
| **P2 核心discretionary** | 慢循环（每日定时） | 黑盒推理 | 可投 ticker + 完整推理，供人工筛选 | **仅前向**纸面 + 复盘，按战绩缩放 |
| **P3 决策辅助** | 快循环（实时） | 行为/咨询 | 行为教练、纪律评分、随问随答的理性建议 | 以行为缺口/override 质量度量 |

### 3.2 LLM 权限边界（修复 D6 + 调和你的诉求）
- 对 **P1 量化信号**：LLM 只能**否决或缩小（veto/shrink），绝不放大**。
- 在 **P2 管道**：LLM **自主发起**discretionary交易候选（黑盒推理允许），但只是**给人工筛选的提案**，经人工门禁 + 风险模型 + 前向战绩缩放——这与"放大量化信号"无关，故无冲突。
- 一致铁律：**不以 LLM 自身的回测认证其discretionary选股**（我知道历史，回测被污染）；P2 仅以前向战绩认证、按战绩缩放。这不是对 LLM 的特殊不信任，而是对一切策略一视同仁。

### 3.3 单次决策数据流
1. 摄取：数字进结构化库，文档进向量库，L1–L7 索引。
2. P1 引擎对当前数据运行**已验证策略** → 候选交易，按**时间归一化期望值**评分，**风险模型预先定仓**（含止损 + 组合风险检查 + 回撤降险）。
3. P2 每日聚合 → 有理由的候选清单。
4. LLM 为候选补定性论点/红队，并检索 Sicong 相似历史交易及结果。
5. 装配提案（交易 + 期望值 + 仓位 + 止损 + 组合风险影响 + 推理 + 行为标记）→ 人工门禁。
6. 执行（纸面 / 经富途 MCP 实盘）→ 写 L7 → 复盘循环评分纪律、回灌 RAG 与发现循环。

## 4. 量化引擎（Python）

### 4.1 策略统一接口
同构模块：`Hypothesis`（为何有 edge）/ `Universe` / `Entry rules` / `Exit rules`（**止损 + 移动止损/目标**）/ `Holding horizon` / `Status`（candidate→probation→promoted→retired）。**必须支持横截面策略**（同时看多标的做排序，如相对动量/热门板块续强）——见 §12。

### 4.2 仓位 — 风险优先、期望值加权
`基础仓位 = 权益 × 单笔风险% / |进场价 − 止损价|`。
- 单笔风险% 起始 0.5–1%。
- **分数 Kelly（封顶 ¼）** 按策略**已证明**的期望值缩放；未证明给最小仓；**带最小样本门槛**，不在 n 极小时放大（修复代码隐患）。
- 波动率归一（ATR）。
- **最小经济仓位**：低于"成本吃掉 edge"的下限不开仓（修复 D11）。

### 4.3 组合风险模型 — 硬性护栏（阻塞）
| 控制项 | 规则（可调） |
|---|---|
| 总热度 | 未平仓风险之和 ≤ ~6% 权益 |
| 单仓上限 | ≤ ~15–20% 权益 |
| 相关/集群上限 | 限制相关标的/板块合计敞口；**集群定义具体化**（行业 GICS + 滚动收益相关阈值二者取严，修复 D12） |
| 最大回撤降险 | 组合回撤达 −X% 时减小仓位、暂停新仓（**按市值实时回撤**，见 §12） |
| 单市场上限 | US/HK 各设上限 |
| FX 敞口 | 跟踪并可选封顶 USD/HKD 集中度 |
| 并发策略上限 | 同时在跑的策略数上限（避免资金/成本碎片化，修复 D11） |
| Regime 分散 | 主动跨 regime 分散（趋势/均值回归/波动），不押单一 regime（修复 D7） |

### 4.4 成本始终内置
见 §2.4。回测/前向每笔含佣金、滑点、点差、市场费、FX、税。

## 5. 策略发现与前向验证循环

### 5.1 生命周期
`IDEA(LLM 提) → SCREEN(回测,仅规则类) → PROBATION(前向纸面,小仓) → PROMOTED(分层实盘,按 edge 缩放,人工批准) → RETIRED(衰减/违规)`。

### 5.2 假设生成
LLM 跨**全 edge 菜单**持续生成假设——趋势/动量、质量、价值、事件驱动、季节性、均值回归、跨资产、情绪等任何有**经济学依据**者，并参考 Sicong 自身有效结果。每个写成 §4.1 模块。

### 5.3 验证闸门（统计诚实）
经济学依据 ✓；**样本外/walk-forward 净成本正期望** ✓；**最小样本**（不在 n=5 晋级）✓；**置信区间为正**（bootstrap，非正态近似，§12）✓；**回撤/路径可接受** ✓；**低相关** ✓；**前向证明** ✓；**年化 R 达标**（时间归一化）✓。

### 5.4 防过拟合
- 仅以 walk-forward/样本外评判。
- **参数节俭**，惩罚复杂重调参。
- **多重检验意识**：提出大量策略 → 相应提高门槛。
- **预注册 + 前向加权**（承认 LLM 提案本身带后视偏差，故重前向轻回测，修复 D10）。
- 双轨：规则类 = 回测 + 前向；discretionary = 仅前向。

### 5.5 部署吞吐（修复 D4）
为避免"验证太慢以致永不上线"：
- **并行前向纸面**多策略同时跑。
- **分层实盘**：通过前向即以极小仓上实盘，置信区间收窄则放大。
- **偏好快认证策略**（更高频者更快达统计显著）。
- 低频策略接受更宽 CI + 更小仓，随证据缩放。

### 5.6 晋级、缩放、淘汰
晋级实盘须 Sicong 批准；资金随 edge 缩放（§4.3 上限内）；**衰减检测**监控滚动期望值，edge 侵蚀/违规即降级或退役。

## 6. RAG 与记忆

### 6.1 三存储
| 存储 | 拥有 | 用于 |
|---|---|---|
| 结构化库(SQLite/Parquet) | 一切数字：**复权后**价格/因子/财务/收益/期望统计/交易记录 | 精确查询、信号、回测、风险 |
| 向量库(ChromaDB) | 一切文档+记忆：财报/纪要/笔记/日志 | 研究与教练语义检索 |
| Vault(Obsidian/MD) | 人类可读规范文件 | Sicong 读写；L3/L5/L7 真源 |

**铁律**：数字永不来自向量库。
**复权铁律**：价格类一律**复权（拆分/分红调整）**后入库，否则止损/期望被污染（修复 D9）。

### 6.2 L1–L7 映射
L1/L2 → 数字进结构化库、文档进向量库（标注 source/date/ticker/market/evidence_level）；L3/L4/L5 → Vault + 嵌入；L6 → 结构化库阈值；L7 → 结构化库（交易记录）**+** 向量库（理由/教训）。

### 6.3 集合
1. 研究语料（财报/纪要/调研/新闻）2. **个人交易记忆（最高价值）**（决策/被拒原因/复盘教训）3. 策略库（假设+测试结果）4. 市场/板块/regime 上下文。

### 6.4 检索管线
混合（语义+BM25）→ 元数据过滤（**date point-in-time**/ticker/market/doc_type/evidence_level）→ re-rank → 预算内装配。**无法解析引用硬失败**；历史分析必须 date 过滤。

### 6.5 最高价值查询
任何新提案前：检索"我自身相似 setup/同标的/同策略历史交易及结果与教训"，驱动 §8 行为教练。

## 7. P2 核心 Discretionary 管道

- **每日定时聚合**：RAG 档案 + 实时行情 + 财经新闻 + 研报。
- **产出**：可投 ticker 排序清单 + **完整推理逻辑**（论点、证据等级、风险、止损建议）。
- **人工筛选**：Sicong 从清单中挑选；选中者进入人工门禁 + 风险模型 + 前向跟踪。
- **认证与缩放**：仅以**前向战绩**认证，按已证明期望值缩放真实资金（与一切策略同纪律）。
- **诚实定位**：黑盒discretionary选股历史上是最不可靠的 edge 源，故按战绩而非信念定仓；人工筛选 + §8 教练为其over-trust的缓冲。

## 8. P3 行为教练与决策辅助

### 8.1 两类
硬性护栏（硬阻塞，§8.2）+ 教练（软性教育，量化代价、提升能力）。

### 8.2 硬性护栏（默认阻塞）
必须止损；禁止亏损加仓；禁止超仓（破 §4.3 上限）；禁止破组合上限；**连亏冷却**（连续 N 次止损/日内亏损阈值后暂停新仓）；**仅已验证策略**（例外需显式留痕 override）。Override 有摩擦且留痕，系统追踪其表现。

### 8.3 行为期望值缺口
`行为缺口 = 计划期望值(R) − 实际实现(R)`，以 R 与 HKD 双报，定位泄漏来源（过早止盈、超止损硬扛、盈后超仓、FOMO）。

### 8.4 override 质量评分（修复 D8）
不一概视偏离为泄漏。每次 override 标注理由并事后评估：
- **合理 override**（有据、留痕、事后被验证正确）→ 正面，沉淀为新认知。
- **冲动 override**（无据、违纪、事后错误）→ 负面，沉淀为盲区。
目标是培养**判断力**而非盲从——系统并非全知，人有时是对的。

### 8.5 决策辅助（随问随答）
Sicong 提交任何交易咨询时，P3 给出**理性、可执行**的建议（基于 RAG + 当前持仓 + 风险模型 + 历史教训），并标注证据等级与风险。

### 8.6 成长追踪
显式度量：行为缺口是否缩小？复发错误是否减少？override 质量是否提升？将"成长为优秀交易者"可量化。

## 12. 工程保真要求（回测/执行不得失真）
以下为**强制实现要求**（源自代码评审，避免回测乐观偏差）：
1. **风险模型并入回测**：回测须经 §4.3 组合风险模型（总热度/单仓/集群/市场/回撤降险），不得无约束并发持仓。
2. **跳空/滑点成交**：止损/目标在跳空穿越时按**更差价**（如止损按 `min(开盘, 止损)`）成交；每笔计滑点。进场按**真实可成交价**（次开/收），非策略任意 `entry`。
3. **移动止损/动态退出**：引擎须支持随价格上移的移动止损与动态退出，以实现"让利润奔跑"。
4. **横截面支持**：策略 `generate` 须能同时接收多标的、做排序（相对动量/热门板块）。
5. **按市值（mark-to-market）回撤**：回撤按逐日浮动权益计，非仅平仓权益曲线。
6. **bootstrap 置信区间**：期望值 CI 用 bootstrap（应对小样本与肥右尾），非正态近似。
7. **复权价**：见 §6.1。
8. **健壮性**：权益为零的除零保护；Kelly/晋级的最小样本门槛；做空可行性按市场规则校验。
